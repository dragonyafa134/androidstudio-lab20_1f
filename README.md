# TODO

Простое консольное приложение на языке Dart, позволяющее добавлять, отмечать выполненными, удалять и просматривать список задач

## Автор

- **Имя:** Дарья
- **Группа:** ИСП 233

## Как запустить проект

1. **Клонируйте репозиторий**  
   ```bash
   git clone <URL вашего репозитория>
   cd todo_app
   ```

2. Убедитесь, что установлен Dart SDK
   ```
   dart --version
   ```
3. Запустите приложение
   ```
   dart run bin/todo_app.dart
   ```
5. Команды
 ```
 add <текст> – добавить новую задачу

list – показать все задачи

done <id> – отметить задачу выполненной

delete <id> – удалить задачу

exit – выйти из программы
```
## Ответы на вопросы

### 1. Чем `final` отличается от `const` в Dart?

| `final` | `const` |
|---------|---------|
| Значение присваивается **один раз** | Значение известно **на этапе компиляции** |
| Определяется **во время выполнения** (runtime) | Является **compile-time константой** |
| Пример: `final now = DateTime.now();` | Пример: `const pi = 3.14159;` |
| Может быть вычислено динамически | Не может зависеть от вычислений в runtime |

### 2. Что означает `String?`?

`String?` — это **nullable-тип**. Переменная с таким типом может хранить либо строку (`String`), либо значение `null`. Это часть механизма **Sound Null Safety** в Dart, который требует явно указывать, может ли переменная быть null.

### 3. Чем `Future` отличается от обычного значения? Что означает `await` с точки зрения потока выполнения?

- **Обычное значение** (например, `String`) доступно синхронно — сразу в момент выполнения кода.
- **`Future<T>`** представляет асинхронную операцию, которая завершится **в будущем** и вернёт результат типа `T` (или ошибку). Пока операция выполняется, поток не блокируется.

- **`await`** приостанавливает выполнение **текущей асинхронной функции** (не блокируя основной поток), позволяя **event loop** обрабатывать другие события (например, таймеры, ввод/вывод). Когда ожидаемый `Future` завершается, выполнение функции возобновляется с полученным результатом.

### 4. Зачем в Dart именованные конструкторы, если в C# есть перегрузка?

- В Dart **нет перегрузки конструкторов** (как в C# или Java). Нельзя создать несколько конструкторов с одинаковым именем, но разными параметрами.
- Вместо этого используются **именованные конструкторы**, которые позволяют создавать объекты разными способами с **явными именами**, делающими код понятнее:
  ```dart
  class Todo {
    Todo.standard(this.title);
    Todo.urgent(this.title) : isUrgent = true;
    Todo.fromJson(Map<String, dynamic> json) : ...
  }
